SQLite-based vulnerability database for the multi-agent C/C++/Python/Go/Lua/Java vulnerability scanner.

Replaces the previous JSON file-based intermediate storage (candidates_df_*.json, candidates_sec_*.json, verified_*.json) with a single SQLite database file (scan.db).

Commands:

1. init - Create or reset the database with all required tables and indexes. Call once at the start of a scan.
   Required: db_path

2. insert - Batch insert candidate vulnerabilities from a scanner.
   Required: db_path, vulnerabilities (JSON array string of vulnerability objects)
   Each object must have: id, source_agent, source_module
   Optional fields: language, framework, analysis_kind, type, cwe, severity, description, file, line_start, line_end, function_name, code_snippet, data_flow, source_kind, sink_kind, sanitizer_checked, evidence_json, rule_id, analysis_backend, pre_validated, cross_module, modules_involved
   id must match: VULN-{DF|SEC}-{CPP|PY|GO|LUA|JAVA|MIX}-{KIND}-{MODULE}-{NNN}
   Examples: VULN-DF-CPP-MEMCPY-IPC-001, VULN-DF-PY-SQLI-SEARCH-001, VULN-SEC-JAVA-CONFIG-HTTPCLIENT-001

3. query - Query vulnerabilities with filters. Returns JSON array of matching rows.
   Required: db_path
   Optional: phase, status, source_module, source_agent, language, analysis_kind, min_confidence, exclude_status, limit, offset, ids (comma-separated IDs)

4. update - Update a single vulnerability's fields (e.g. verification results).
   Required: db_path, id, fields (JSON object string with fields to update)
   Automatically sets phase='verified' and updated_at.

5. batch-update - Update multiple vulnerabilities at once.
   Required: db_path, updates (JSON array string of {id, fields} objects)

6. dedup - Deduplicate candidates by (file, line_start, function_name, type). Keeps the highest-severity entry, marks others as dedup_kept=0. Merges source_agent info.
   Required: db_path
   Returns dedup statistics.

7. stats - Get aggregated statistics.
   Required: db_path
   Optional: phase (filter by phase)
   Returns counts by status, severity, module, language, analysis kind, and totals.

8. log - Record an agent execution event.
   Required: db_path, agent_name
   Optional: module_name, phase, status, message, item_count

9. export-json - Export query results to a JSON file (for debugging/compatibility).
   Required: db_path, output
   Optional: phase, status, source_module, min_confidence, exclude_status

10. work-add - Add scan work items for large-project sharded scanning.
    Required: db_path, work_items (JSON array string)
    Each item should include: id, agent_name, profile, round, pass_id, pass_kind, shard_type, language, module_id, source_module, files, priority

11. work-query - Query scan work items.
    Required: db_path
    Optional: status, agent_name, source_module, module_id, language, profile, round, pass_id, pass_kind, shard_type, limit

12. work-claim - Claim pending work items in priority order.
    Required: db_path
    Optional: agent_name, language, profile, round, pass_id, pass_kind, limit
    Marks returned items as running and increments attempts.

13. work-complete - Mark one work item as success.
    Required: db_path, id
    Optional: finding_count or item_count

14. work-fail - Mark one work item as failed.
    Required: db_path, id
    Optional: message

15. work-requeue - Move running/failed work items back to pending for resume.
    Required: db_path
    Optional: id, agent_name

16. work-stats - Get work item status counts.
    Required: db_path
    Optional: agent_name, source_module, module_id, language, profile, round, pass_id, pass_kind

17. coverage-add - Add coverage ledger records after work item completion.
    Required: db_path, coverage_items (JSON array string)
    Each item should include: agent_name, work_item_id, profile, round, pass_id, pass_kind, source_module/module_id, language, shard_type, files, coverage_status, findings_count.
    coverage_status: complete, partial, blocked, shallow, expansion_needed

18. coverage-query - Query coverage ledger records.
    Required: db_path
    Optional: agent_name, work_item_id, source_module, module_id, language, profile, round, pass_id, pass_kind, shard_type, coverage_status, limit

19. coverage-stats - Get coverage ledger status counts.
    Required: db_path
    Optional: agent_name, source_module, module_id, language, profile, round, pass_id, pass_kind, shard_type, coverage_status

Parameters:
- db_path: Absolute path to the SQLite database file (e.g. {CONTEXT_DIR}/scan.db)
- command: The operation to perform
- vulnerabilities: JSON array string for insert command
- id: Vulnerability ID for update command
- fields: JSON object string for update command
- updates: JSON array of {id, fields} for batch-update
- phase: Filter by phase (candidate/verified)
- status: Filter by status (CONFIRMED/LIKELY/POSSIBLE/FALSE_POSITIVE)
- source_module: Filter by module name
- module_id: Filter by stable module ID
- source_agent: Filter by source agent
- language: Filter by language (c_cpp/python/go/lua/java)
- analysis_kind: Filter by analysis kind (dataflow/authn/authz/crypto/config/secret/framework_misuse)
- min_confidence: Minimum confidence score filter
- exclude_status: Exclude vulnerabilities with this status
- limit: Max number of results to return
- offset: Pagination offset
- ids: Comma-separated list of vulnerability IDs to query
- agent_name: Agent name for log command
- profile: Scan profile (quick/standard/deep/paranoid)
- round: Scan round number
- pass_id: Independent scan pass number
- pass_kind: Independent pass kind (primary/sink_to_source/negative_review/cross_module/disagreement_review)
- work_item_id: Work item ID for coverage records
- coverage_status: Coverage status for coverage-query
- module_name: Module name for log command
- message: Log message
- item_count: Number of items processed (for log)
- output: Output file path for export-json
- work_items: JSON array of scan work items for work-add
- coverage_items: JSON array of coverage ledger items for coverage-add
- shard_type: Work item shard type (entrypoint_slice/sink_slice/module_sweep/expansion_slice/cross_module_slice)
- finding_count: Number of findings produced by a completed work item
